# Анализ продаж интернет-магазина (Olist E-commerce)

## Цель проекта
Провести анализ продаж интернет-магазина для оценки ключевых бизнес-метрик и выявления драйверов выручки.  
Проект демонстрирует навыки SQL, включая оконные функции, агрегаты и аналитические запросы.

## Данные
Данные взяты с [Kaggle: Olist E-commerce](https://www.kaggle.com/olistbr/brazilian-ecommerce).  
Основные таблицы:

- `orders` – заказы  
- `order_items` – товары в заказах  
- `products` – информация о товарах  
- `customers` – клиенты  
- `payments` – информация о платежах  

## Технологии
- PostgreSQL  
- SQL
- DBeaver для работы с базой  

## Структура репозитория
ecommerce-sales-analysis/
│
├── sql/
│ ├── 01_basic_analysis.sql -- базовые метрики и топ товаров
│ ├── 02_customer_analysis.sql -- анализ клиентов и повторных покупок
│ ├── 03_retention_analysis.sql -- когортный анализ и retention
│ └── 04_window_functions.sql -- оконные функции: ранжирование, кумулятивная выручка, средний


## Основные задачи и результаты

1. Рассчитана общая выручка и выручка по месяцам  
2. Определены топ-товары и топ-категории по выручке  
3. Рассчитан средний чек и среднее количество товаров в заказе  
4. Проанализировано среднее время доставки (12 дней)  
5. Проанализированы повторные покупки и доля повторных клиентов  
6. Проведён когортный анализ по месяцу первой покупки  
7. Использованы оконные функции для:
   - ранжирования товаров в категории (RANK)  
   - кумулятивной выручки по месяцам (SUM OVER)  
   - среднего чека по клиенту (AVG OVER)  
   - предыдущей покупки клиента (LAG)  
   - номера заказа клиента (ROW_NUMBER)  
   - процента выручки каждого заказа от общей выручки клиента  

## Как использовать
1. Скачайте датасет с Kaggle и загрузите таблицы в PostgreSQL  
2. Откройте файлы в папке `sql/` в DBeaver и выполните их по порядку
